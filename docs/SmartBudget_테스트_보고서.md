## 테스트 환경 및 판정 기준

### 테스트 환경
- Frontend: React(로컬) / URL: http://localhost:3000
- Backend: Spring Boot(로컬) / URL: http://localhost:8080
- AI Server: Python(FastAPI, 로컬) / URL: http://localhost:8000
- DB: PostgreSQL (schema: projectdb)
- 테스트 계정: userId=1 (테스트용 계정)

### PASS/FAIL 판정 기준(공통)
- PASS: 케이스의 핵심 조건 충족(예: 화면 이동/요소 표시 + API 2xx + 필요한 경우 DB 반영 확인)
- FAIL: 4xx/5xx 오류, 화면 미반영, DB 미반영, 또는 예외 발생 시 사용자 안내 미노출
- BLOCKED: 외부 의존(ML 서버/DB) 장애로 테스트 진행 불가

## 1. 기능 테스트 개요

### 기능 테스트란

- **화면·기능 단위**로 사용자 시나리오가 명세대로 동작하는지 검증하는 테스트
- 랜딩·로그인·대시보드·거래 추가·가계부·예산·목표·추천·AI 리포트·마이페이지·네비게이션 등 **메뉴/경로별**로 케이스 구성
- 사전 조건·테스트 데이터·예상 결과를 정리하여 수동 또는 반자동 실행

### 기능 테스트 목적

| 목적 | 설명 |
|------|------|
| 요구사항 충족 확인 | 각 화면·기능이 기획/명세대로 동작하는지 검증 |
| 회귀 방지 | 배포 전 주요 플로우 점검으로 결함 조기 발견 |
| 테스트 커버리지 관리 | 분류별 케이스 목록으로 누락 구간 파악 |

## 4. 통합 테스트 개요

- **전체 시스템**이 올바르게 동작하는지 확인하는 테스트 단계
- **기능 테스트**로 점검한 화면·플로우를 연계하여 E2E 검증
- **FE ↔ Spring ↔ Python(OCR/ml-server)** 간 데이터 교환·API 호출·DB 연동 검증
