# POST /api/llm/budget-insight 404 로그 설명

## 이 로그가 뭔가요?

- **누가 요청하나:** Spring 백엔드(Java)가 **예산 인사이트**를 받아오려고 할 때, 내부적으로 **ML 서버(Python)** 에 `POST /api/llm/budget-insight` 를 호출합니다.
- **어디서 쓰이나:**  
  예산/목표 화면에서 쓰는 **예산 인사이트**(저축 추천, 예산 대비 사용, 추세·이번 달 전망 등)를 만들 때 사용합니다.  
  거래추가 페이지로 이동할 때도 앱이 같이 돌아가면서 이 API가 호출될 수 있습니다.
- **404 의미:**  
  그 요청이 **지금 돌아가고 있는 Python 서버**에 도달했는데, 그 서버에는 **`/api/llm/budget-insight` 라우트가 없어서** “Not Found”(404)가 난 상태입니다.

즉, **“예산 인사이트용 API를 Python 서버에 요청했는데, 그 서버가 해당 경로를 제공하지 않아서 404”** 라는 뜻입니다.

## 왜 404가 나나요?

- **`/api/llm/budget-insight` 를 제공하는 쪽은 `ml-server`(SmartBudget ML 서버)입니다.**  
  `smartbudget/ml-server/app/main.py` 에 이 경로가 정의되어 있습니다.
- 그런데 지금 터미널에서 돌린 **`python run.py` 가 `ml-server`가 아니라 `OCR_receipts` 쪽**이면,  
  OCR 서버에는 이 경로가 없기 때문에 **같은 포트(8000)로 들어온 요청이 404**를 반환합니다.
- 또는 **포트가 8000이 아닌 다른 서버**로 요청이 가도록 설정돼 있으면, 그 서버에 라우트가 없을 때도 404가 납니다.

정리하면:

- **404 = “예산 인사이트용 API를 호출했는데, 받는 쪽 Python 서버에 그 API가 없음”**
- 대표적인 원인: **지금 실행 중인 Python 서버가 `ml-server`가 아니라 OCR 서버이거나, 8000번에서 ML 서버가 안 떠 있는 경우.**

## 어떻게 하면 되나요?

1. **예산 인사이트 기능을 쓰려면**
   - **ML 서버**를 띄워야 합니다.  
     `smartbudget/ml-server` 폴더에서:
     ```bash
     python run.py
     ```
   - Spring 설정에서 ML 서버 주소가 `http://localhost:8000` (또는 실제 ML 서버 주소)로 되어 있는지 확인하세요.
2. **예산 인사이트는 안 써도 된다면**
   - 이 404는 **기능 누락**이지, 거래추가 자체를 막는 오류는 아닙니다.  
     로그만 찍히고, 예산 인사이트만 비어 있을 수 있습니다.
3. **OCR도 쓰고 ML도 써야 한다면**
   - 두 서버가 **같은 포트(8000)** 를 쓰면 한 번에 하나만 실행됩니다.  
     OCR 서버를 다른 포트(예: 8001)로 띄우거나, ML 서버를 8000으로 두고 OCR은 8001로 두는 식으로 포트를 나눠서 두 개 모두 실행해야 합니다.

요약하면, **“거래추가를 누르면 뜨는 로그” = 예산 인사이트를 위해 Spring이 Python ML 서버를 호출했는데, 지금 떠 있는 Python 서버가 그 API를 제공하지 않아서 404가 난 것**입니다.
